# Android 应用内更新使用指南

## 📱 功能说明

应用内更新功能允许用户在APP内直接下载并安装新版本的APK，无需跳转到外部浏览器或应用商店。

### ✨ 特性

- ✅ **自动检查更新**：应用启动时自动检查更新（每天最多一次）
- ✅ **进度显示**：实时显示下载进度
- ✅ **自动安装**：下载完成后自动打开安装界面
- ✅ **强制更新**：支持强制更新模式
- ✅ **稍后提醒**：可选更新可以稍后安装（24小时后再次提醒）

---

## 🚀 使用方式

### 1. 自动检查更新

应用启动时会自动检查更新（每天最多一次），如果有新版本会弹出更新提示。

```typescript
// 在 diary-list/index.tsx 中使用
import { useAppUpdate } from '@/hooks/useAppUpdate';

const { checkAutoUpdate, UpdateModal } = useAppUpdate();

// 在页面加载时检查更新
useFocusEffect(
  useCallback(() => {
    checkAppUpdate();
  }, [])
);

// 渲染进度Modal
return (
  <View>
    {/* ... */}
    <UpdateModal />
  </View>
);
```

### 2. 手动检查更新

可以在设置页面添加"检查更新"按钮：

```typescript
const { checkAndShowUpdate, UpdateModal } = useAppUpdate();

const handleCheckUpdate = async () => {
  const hasUpdate = await checkAndShowUpdate();
  if (!hasUpdate) {
    Alert.alert('提示', '已是最新版本');
  }
};

// UI
<TouchableOpacity onPress={handleCheckUpdate}>
  <Text>检查更新</Text>
</TouchableOpacity>
```

## 🔧 技术实现

### 1. 核心Hook

**文件**：`client/hooks/useAppUpdate.ts`

**主要功能**：

| 函数 | 说明 |
|------|------|
| `checkForUpdate()` | 检查是否有更新 |
| `checkAndShowUpdate()` | 检查并显示更新对话框 |
| `checkAutoUpdate()` | 自动检查更新（每天一次） |
| `downloadAndInstallAPK()` | 下载并安装APK |
| `skipUpdate()` | 跳过更新（24小时后再次提醒） |
| `UpdateModal` | 下载进度Modal组件 |

### 2. 依赖库

| 库 | 版本 | 用途 |
|-----|------|------|
| `expo-file-system` | ~19.0.21 | 下载APK文件 |
| `expo-intent-launcher` | 55.0.12 | 打开APK安装界面 |
| `@react-native-async-storage/async-storage` | 2.2.0 | 存储检查时间和跳过状态 |

### 3. 权限配置

**文件**：`client/app.config.ts`

**必需权限**：

```json
{
  "permissions": [
    "INTERNET",                    // 网络权限（下载APK）
    "READ_EXTERNAL_STORAGE",       // 读取存储
    "WRITE_EXTERNAL_STORAGE",      // 写入存储
    "REQUEST_INSTALL_PACKAGES"     // 安装应用权限
  ]
}
```

## 📦 后端配置

### 创建版本记录

在Web工具（`http://localhost:9092/version-manager`）中创建版本记录：

**必填项：**
- 版本号（如：1.0.1）
- 构建号（如：101）
- 平台（android）
- 更新说明
- 更新类型（可选/强制）

**可选项：**
- 更新包URL（APK下载地址）

**示例：**

```
版本号: 1.0.1
构建号: 101
平台: Android
更新类型: 可选更新
更新说明:
1. 修复了登录失败的bug
2. 优化了启动速度
3. 改进了UI界面

更新包URL: https://cdn.example.com/app-v1.0.1.apk
```

### APK上传

1. 构建APK：
```bash
cd client
eas build --platform android --profile production
```

2. 上传到CDN（如阿里云OSS、腾讯云COS等）

3. 获取下载URL

4. 在Web工具中填写更新包URL

5. 激活版本

### 场景1：小Bug修复（可选更新）

```json
{
  "version": "1.0.1",
  "force_update": false,
  "release_notes": "修复了登录失败的bug"
}
```

**用户行为**：
- 弹出更新提示
- 用户可以选择"下次继续"
- 用户点击"立即更新"后下载并安装

### 场景2：安全漏洞修复（强制更新）

```json
{
  "version": "1.0.2",
  "force_update": true,
  "release_notes": "重要安全更新！修复了严重的安全漏洞，请立即更新"
}
```

**用户行为**：
- 弹出强制更新提示
- 必须点击"立即更新"
- 无法跳过或关闭

### 场景3：重大功能更新（可选更新）

```json
{
  "version": "1.1.0",
  "force_update": false,

}
```

**用户行为**：
- 弹出更新提示
- 用户可以选择"下次继续"或"立即更新"
- 用户点击"立即更新"后下载并安装

## ⚠️ 注意事项

### 1. APK文件大小

- **建议**：APK文件大小不超过50MB
- **原因**：大文件下载时间长，用户体验差
- **解决方案**：使用APK分包（Android App Bundle）

### 2. 下载URL

- **要求**：必须是公网可访问的HTTPS地址
- **推荐**：使用CDN加速（阿里云OSS、腾讯云COS等）
- **注意**：URL不能包含中文字符

### 3. 网络环境

- **WiFi**：自动下载
- **4G/5G**：可以下载，但建议提示用户
- **无网络**：显示"网络错误"，稍后重试

### 4. 存储空间

- **检查**：下载前检查设备存储空间
- **提示**：如果空间不足，提示用户清理存储
- **缓存**：下载完成后可以删除旧版本的APK

### 5. Android版本

- **Android 7.0+**：支持应用内更新
- **Android 6.0及以下**：需要手动安装
- **建议**：提示用户升级Android系统

### 6. 权限要求

用户首次更新时需要授权：
- 允许安装未知来源应用
- 允许访问存储

**注意**：Android 8.0+需要在设置中手动开启"允许安装未知来源应用"权限。

## 🐛 常见问题

### Q1: 下载失败怎么办？

**A**: 检查以下几点：
1. 网络连接是否正常
2. APK下载URL是否可访问
3. 设备存储空间是否充足
4. 是否有下载权限

### Q2: 下载完成后没有打开安装界面？

**A**: 检查以下几点：
1. APK文件是否下载成功
2. 是否有安装权限（REQUEST_INSTALL_PACKAGES）
3. Android版本是否支持（7.0+）

### Q3: 强制更新用户跳不过去怎么办？

**A**: 强制更新是设计如此，用户必须更新才能继续使用。如果更新有问题：
1. 使用Web工具修改版本为非强制更新
2. 或者修复更新包并重新上传

### Q4: 用户不想更新怎么办？

**A**: 如果是可选更新，用户可以点击"下次继续"，24小时后再次提醒。

### Q5: 如何测试更新功能？

**A**: 方法如下：

1. 在Web工具创建版本记录
2. 确保版本号大于当前版本（1.0.0 → 1.0.1）
3. 填写APK下载URL
4. 激活版本
5. 重启应用，检查是否弹出更新提示
6. 点击"立即更新"，测试下载和安装

### Q6: iOS不支持应用内更新？

**A**: 是的，iOS不支持应用内更新APK，需要：
1. 使用App Store的SKStoreProductViewController
2. 或者跳转到App Store更新页面
