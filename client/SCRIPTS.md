# 脚本使用说明

## 概览

本项目包含以下实用脚本，用于开发、构建和部署。

## 脚本列表

### build-apk.sh

**功能**：构建 Android APK 文件

**使用方法**：
```bash
./build-apk.sh
```

**说明**：
- 自动检查构建环境
- 检查 Expo 登录状态
- 支持选择构建方式（EAS Build 或本地构建）
- 自动配置生产环境

---

### check.sh

**功能**：检查开发环境配置

**使用方法**：
```bash
./check.sh
```

**检查内容**：
- Node.js 版本
- 依赖安装状态
- 配置文件完整性
- 环境变量配置
- TypeScript 编译检查

**适用场景**：
- 首次设置开发环境
- 遇到问题时排查
- 准备构建前检查

---

### clean.sh

**功能**：清理缓存和临时文件

**使用方法**：
```bash
./clean.sh
```

**清理内容**：
- node_modules/
- .expo/ 缓存
- 构建输出文件
- 临时文件

**适用场景**：
- 依赖安装失败时
- 遇到缓存问题时
- 准备构建前清理

---

### deploy.sh

**功能**：部署到生产环境

**使用方法**：
```bash
./deploy.sh
```

**说明**：
- 检查环境配置
- 执行构建流程
- 部署到目标环境

**适用场景**：
- 自动化部署流程
- CI/CD 集成

## 使用建议

### 首次设置

```bash
# 1. 检查环境
./check.sh

# 2. 安装依赖（如果需要）
npm install

# 3. 启动开发服务器
npm start
```

### 构建APK

```bash
# 1. 检查环境
./check.sh

# 2. 清理缓存（可选）
./clean.sh

# 3. 构建APK
./build-apk.sh
```

### 问题排查

```bash
# 1. 清理缓存
./clean.sh

# 2. 检查环境
./check.sh

# 3. 查看详细日志
npm start
```

## 注意事项

1. **权限**：确保脚本有执行权限
   ```bash
   chmod +x *.sh
   ```

2. **环境**：所有脚本必须在项目根目录运行

3. **网络**：构建过程需要稳定的网络连接

4. **登录**：构建APK前需先登录 Expo 账户
   ```bash
   npx expo login
   ```
