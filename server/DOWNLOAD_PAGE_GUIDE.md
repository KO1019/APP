# 下载页面设计说明

## 🎨 设计理念

参考QQ下载页面的简洁风格，打造极简、实用的应用下载页面。

### 设计特点

#### 1. 极简主义
- ✅ 大Logo居中展示
- ✅ 简洁的版本信息
- ✅ 清晰的下载列表
- ✅ 无过度装饰

#### 2. 真实实用
- ✅ 直接下载链接
- ✅ 版本信息和更新内容
- ✅ 平台适配说明
- ✅ 明确的状态提示

#### 3. 状态管理
- ✅ 加载中状态
- ✅ 下载可用状态
- ✅ 暂未开放状态
- ✅ 错误处理

---

## 📱 页面结构

### 1. Logo区域
- 大尺寸Emoji图标（80px）
- 应用名称（48px）
- 品牌口号

### 2. 版本信息
- 版本号（带颜色突出）
- 更新时间
- 更新内容说明

### 3. 下载列表
- 平台名称
- 系统要求说明
- 下载按钮或"暂未开放"提示

### 4. 底部链接
- 用户协议
- 隐私政策
- 联系我们
- 帮助中心

---

## 🎯 功能说明

### 1. 数据获取
```javascript
async function fetchDownloadInfo() {
    const response = await fetch('/api/v1/app/download');
    const data = await response.json();
}
```

### 2. 状态判断
- **有下载链接**：显示下载按钮
- **无下载链接**：显示"暂未开放"

### 3. 下载行为
- **Android**：直接下载（`download`属性）
- **iOS**：跳转到App Store（`target="_blank"`）

## 🔧 技术实现

### 1. 加载状态
```javascript
// 初始显示加载状态
<div id="loading" class="loading">加载中</div>
```

### 2. 动态生成下载列表
```javascript
// Android
if (data.android && data.android.url) {
    // 显示下载按钮
} else {
    // 显示"暂未开放"
}

// iOS
if (data.ios && data.ios.url) {
    // 显示下载按钮
} else {
    // 显示"暂未开放"
}
```

### 3. 错误处理
```javascript
try {
    // 获取数据
} catch (error) {
    // 显示暂未开放状态
}
```

## 📊 状态说明

### 加载中
- 显示"加载中..."（带动画点）
- 隐藏版本信息和下载列表

### 下载可用
- 显示版本信息
- 显示下载按钮
- 按钮可点击

### 暂未开放
- 显示版本信息
- 显示"暂未开放"文本
- 按钮替换为不可点击文本

### 错误状态
- 显示版本信息（默认值）
- 显示"暂未开放"文本
- 所有平台都不可下载
